## EC2 Auto Scaling

### 1. 소개
- 애플리케이션을 모니터링하고 용량을 자동으로 조정하는 서비스
- 그룹의 최소, 최대 인스턴스 숫자 유지
- 여러 가용 영역에 인스턴스를 분산


- 스케일링 정책
  - CPU 부하에 따른 인스턴스 개수 조절
  - 특정 시간에 인스턴스 개수 조절
  - etc ...

### 2. 구성
- 시작 구성, 시작 템플릿
  - EC2 유형, 크기
  - AMI, 보안 그룹, 키, IAM 역할
  - 유저 데이터 (인스턴스 실행 시 실행 스크립트)
  - 기타 설정


- 인스턴스 상태 모니터링
  - CPU 점유율
  - 인스턴스 헬스 체크


- 인스턴스 종료 정책
  - 기본
    - 가장 오래된 시작 템플릿을 가진 인스턴스를 종료
    - 다음 과금 시간에 가장 가까운 인스턴스 종료
  - 커스텀
